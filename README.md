# Urban Vehicle Request and Management System

A backend service for managing urban ride requests, drivers, vehicles, and user roles.

This project was developed as part of a Database Management Systems course. It focuses on structured data modeling, role-based access control, request management, and backend API design.

## Features

- User authentication with JWT
- Role-based access control
- Passenger, driver, coordinator, and admin roles
- Ride request creation and management
- Driver and vehicle verification
- Trip status tracking
- MongoDB data modeling with Mongoose
- RESTful API structure

## Technologies

- Node.js
- Express.js
- MongoDB
- Mongoose
- JWT
- JavaScript

## Main Entities

- User
- Driver
- Vehicle
- Request
- Trip

## Project Purpose

The main purpose of this project is to demonstrate how a ride request and vehicle management system can be designed using database management principles and a modular backend architecture.
